php

  • strpos($url, 'api/')注意如果要使用strpos来作为if的判断条件的话，可能会子串刚好在被索引串的开头位置，也就是返回结果为数值0，所以if(strpos($url, 'api/'))是会出现逻辑问题的，应该使用if(strpos($url, 'api/')!==false)
  • array_push($a, $b)，如果$b是数组则会把整个$b当最后的键的值，如果要拼接，还是使用array_merge比较好，也可以直接在数组后面通过+=实现拼凑
  • Guzzle php控制器请求的Cookie无法正常提交（每次的访问都是一次新的连接sessionid不断变换），可以直接用浏览器访问（会自动添加sessionid,访问的连接不断）
  • $_POST能够处理的是表单的数据类型（即有明显的键值对）,如果发现$_POST无法正常获取post数据，可以使用strval(file_get_contents("php://input"))获取原生字符串再处理, 或者$GLOBALS['HTTP_RAW_POST_DATA']，
  • 相同域名下的url访问才会自动发送phpsessid
  • ORM 即对象关系映射，（数据表又叫关系表，这里的关系指数据表）php用Doctrine(ORM),通过php对象轻松访问数据库信息
  • notice级别的错误可以通过error_reporting(E_ALL&~E_NOTICE);来取消提示
  • array_values(array_unique($group_ids))去重并重排键值
  • 如果要将某个具体的键值对插入某个数组中，可以通过splice+array_merge实现，如下
  • list($a, $b) = array($aa, $bb); 同时赋值$a,$b
  • session_id 获取当前会话id
  • mixed call_user_func_array ( callable $callback , array $param_arr ) 将第二个数组参数作为回调函数的参数调用，返回回调函数的结果

  • 图片的输出

  • 数组之间相加，相当于array_push()

  • 关于批量删除指定前缀的文件，可以使用glob函数进行

  • 关于请求发起，有3种方式curl stream_context_create fsockopen

PCRE Perl语言兼容正则表达式

  • 分隔符： / # ~ { }, 可以使用preg_quote()进行转义
  • 元字符：
    • \ 转义或者编码控制
    • ^ 目标开始位置/多行模式下的行首
    • $ 目标结束位置/多行模式下的行尾
    • . 匹配换行符外的任何字符
    • [ ] 字符类定义,在其下的第一个^表示字符类取反
    • - 表示字符范围
    • | 可选分支
    • ( ) 子组定义
    • ? 量词，表示0次或1次匹配 等价{0,1},加在其他量词后面可以改变贪婪特性,量词都是默认贪婪-尽可能多得匹配
    • * 量词，表示0次或多次匹配 等价{0,}
    • + 量词，表示1次或多次匹配 等价{1,}
    • { } 自定义量词标记
  • 转义序列：（模式使用PCRE_EXTENDED编译则空白字符(除了字符类中的)和未转义的#到行末的所有字符都会被忽略，这种时候如果要使用空白字符或#需要转义）
    • \cx “contorl-x”, x是任意字符
    • \f 换页(0x0C)
    • \n 换行(0x0A)
    • \p{xx} 符合xx属性的字符,属性参考
    • \P(xx) 不符合xx属性的字符 (使用属性匹配需要搜索，速度不快)
    • \r 回车(0x0D)
    • \t 水平制表符(0x09)
      (描述特定字符类)
    • \d 任意十进制数字 \D 任意非十进制数字
    • \h 任意水平空白字符 \H 任意非水平空白字符
    • \s 任意空白字符 \S任意非空白字符
    • \v 任意垂直空白字符 \V 任意非垂直空白字符
    • \w 任意单词字符 \W 任意非单词字符
      (转义作简单断言，注意不能出现在字符类中)
    • \b 单词边界 \B 非单词边界 (字符类中\b表示退格符)
    • \A 目标开始位置(独立于多行模式)
    • \Z 目标结束位置或结束处的换行符(独立于多行模式)
    • \z 目标结束位置(独立于多行模式)
    • \G 在目标中首次匹配位置

SMTP协议、POP3协议

  • 站点配置相应的SMTP协议[Simple Mail Transfer Protocol]以使用对应SMTP服务器的验证功能
  • POP3协议[Post Office Protocol 3]则是对应服务器的接受邮件功能，用来查看邮件信息
  • 现在的服务器的SMTP协议有验证功能，如果发送的FROM字段信息和真实账号信息不一致会拒绝发送(变成垃圾邮件)
  • IMAP协议[Internet Mail Access Protocol]交互式邮件存取，在POP3基础上的操作可以反馈到邮件服务器上

git

  • 对于比较靠后的origin/master远程分支，可以通过直接git push origin master将远程的master提前到当前本地所在分支

mysql

  • 有效的查询条件能够正确调用数据表的索引字段(B+树)，没有深思熟虑的查询条件将会造成全表扫描，降低速度
  • 如何建立索引可以参考网上一些博客,要注意过多的索引会增加空间的负载
  • 为了利用索引加快查询速度，在使用大于、小于符号的时候最好有相应的最大值、最小值来形成一个范围
  • 为了有效利用索引，对于索引列在使用like查询的时候遵循最左前缀where ss like 'abc%'
  • 将where语句下的 or 拆分为两组查询语句之后在简单合并速度会快点
  • …

网站高并发的处理

  • 反向代理，利用一个代理服务器来处理外部的请求，分发给多台服务器，降低单个服务器压力
  • CDN找离客户端最近的服务器站点
  • 资源、应用分离，页面请求的静态资源单独存放在一个资源服务器与数据请求的服务器分离，降低服务器负载
  • 缓存框架redis\memcache
