Vue.js随记

2020-02-09

计算属性computed:{}

  • 将计算的数据缓存下来，数据没改变的时候调用缓存数据，提高效能，比method每次重新计算要好一点

v-show/v-if

  • v-show 只是简单的改变display: none; 而v-if 是改变了dom结构的

v-html

  • 正常显示变量的内部标签内容，不再自动转义<div v-html="content"></div>

v-for

  • 可以使用in 或者 of 来作为分隔符
  • v-for也可以用于object的遍历，遍历(value, key, index) index从0开始(对象的遍历是按照Object.keys()的结果遍历)
  • 如果在data改变需要重新渲染的时候默认不会保存子元素的状态(比如表单的输入)，如果有保存状态的需要可以通过:key="item.id"绑定key值来重用元素对于template组件使用v-for的时候key是必须的

    响应式限制(都是由于js的限制)

  • 对于vm.items[index] = newValue 和 vm.items.length = newLen都不是响应式的
    • 对于vm.items[index]的修改，为了满足响应式，可以通过Vue.set(vm.items, index, newValue)(等价于实例方法vm.$set(vm.items, index, newValue))或者使用变异方法vm.items.splice(index, 1, newValue)实现,也可以通过
    • 对于vm.items.length的修改通常是为了缩短数组长度，可以通过vm.items.splice(newLen)实现截断
  • 属性的添加删除
    • 只有在初始化时候的数据是响应式的，不能动态天津爱，只能通过Vue.set(obj, key, value) 向嵌套对象天津爱响应式属性
    • 如果需要添加多个响应式属性,应当使用一个新对象，而不是引用旧对象

v-on事件

  • 时间修饰符，可以在标签内定义事件触发的时候附加的事件(可以串联，代码顺序与串联顺序一致)，比如常见的避免默认事件

    • stop 阻止事件继续传播
    • prevent 阻止默认事件触发
    • capture 使用捕获模式
    • self 只有在event.target触发对象是当前元素时候才会触发，即事件不是从内部元素触发的
    • once 只触发一次，被触发后自动移除触发的函数
    • passive 表示函数内部永远不会调用preventDeafualt()，否则会忽略它并抛错

  • 按键别名`@keyup.enter=”fun”`

    • enter
    • tab
    • delete (捕获“删除”和“退格”键)
    • esc
    • space
    • up
    • down
    • left
    • right
    • 可以通过Vue.config.keyCodes.f1 = 112自定义按键修饰符别名
    • 在2.5.0之后支持按键名识别<input @keyup.page-down="onPageDown">识别event.key为PageDown时候触发
    • 在ie下的按键key值有些不同，为了兼容，尽量使用内置别名是首选
    • 系统修饰键(可与其他按键一起按)`@keyup.alt.67=”clear”//alt+c与常规按键不同，在keyup事件中，系统修饰键必须处于按下的状态，释放其他按键才能触发@keyup.ctrl单单释放ctrl也不会触发事件(如果需要监听ctrl的keyup事件需要使用ctrl`的keyCode)
      • ctrl
      • alt
      • shift
      • meta MAC的command键，windows的菜单键
      • (2.5.0)可以通过添加.exact精确修饰触发的事件(精确指定系统修饰键按下触发`@click.exact=”fun”//不按下任何系统修饰键的时候才触发@click.ctrl.exact=”fun”//只在按下ctrl才触发(同时按下ctrl和alt在click也不触发)`
    • 鼠标修饰键
      • left
      • right
      • middle
  • 子组件的事件触发函数可以通过$emit('parent') 来触发父级组件上的事件(需要父级组件监听事件@parent="fun")

v-model

  • 用在input标签实现数据双向绑定，基于oninput 事件监听(onchange监听是在失去焦点的时候才触发判断，而oninput是在输入的时候就触发)
  • 修饰符
    • v-bind.lazy="msg"惰性同步，使用onchange进行数据同步
    • v-bind.number="age" 自动将用户的输入转化为数值类型(不再是数字的字符串)
    • v-bind.trim="msg"自动过滤用户输入的首尾空白字符

Vue.component(‘’, {}) 组件化

  • Vue.component()是全局注册组件
  • data必须是一个函数返回单独的拷贝，不然变量会被共享
  • 对子组件双向绑定数据(v-model)，需要在子组件中将双向绑定拆分为v-bind:value v-on:input，并定义输入参数value，具体如下

  • 父级元素如果要在子组件的标签中间插入html文本,那么在子组件的相应<slot></slot>位置就会出现父级插入的html文本

  • 通过is属性可以实现动态子组件的加载同时避免子组件被提升到外部

注意

  • 避免v-for/v-if一起使用参考v-for 比 v-if 有更高的优先级

    • 对于条件过滤，应当使用computed过滤后的数组进行循环而不是用v-for v-if组合，要避免非显示的节点数据改变的时候重新渲染,同时解耦便于维护

  • 调用组件的时候申明的变量注意id不能和原组件名一样[Vue warn]: Do not use built-in or reserved HTML elements as component id: dialog

  • 在组件中的import导入必须放在script头部，不能放入export default的内容中去

  • 对于动态require引用第三方库或文件，直接使用路径变量是不行的，参考Nodejs文档说明

  • require 与 import 的不同,nodejs文档说明

  • 使用vue-markdown-loader/lib/markdown-compiler -> vue-loader 可以将md文件转化后的HTML当做标签直接插入参考

  • 动态注册组件

    • 需要安装并配置.babelrc的插件@babel/plugin-syntax-dynamic-import， 同时为了读取markdown内容的图片，记得配置webpack的图片loader

  • 注意v-bind绑定class的格式是在双引号内部写数组<div :class="['dialog, getItemClass(item)]"></div>

  • webpack下的resolve属性用来修改import和require的路径选项，文档

  • 在beforeCreate()中无法读取到$props传入的数据，只有在created()及其以后的声明周期函数中才能使用到

  • 对于ref属性，如果与v-for同时使用那么this.$refs.A会变成数组形式，同时官方是不推荐对ref使用绑定的

  • 在webpack4下模拟接口,对于接口的配置对象app实际上是一个express实例(文档)，参数的req对象参考nodejs文档

    对于post请求的body对象，可以参考文档进行配置
