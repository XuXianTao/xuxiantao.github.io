Deepin系统使用随记

2018-07-29

systemctl list-units 显示所有已安装服务
ststemctl status apache2
systemctl start mysql
apacha 项目目录/var/www/html
apache2 debian路由sudo a2enmod rewrite
服务器重启 systemctl restart apache2
apache2 log日志路径sudo vi /var/log/apache2/error.log

如果配置了apache2的https强制跳转，要注意对应的虚拟站点的端口要从80修改为443,因为https端口443,普通http端口80

mysql -uroot -proot
mysql>select * from user limit 1\G 结果显示成一列
mysql>show create database mysql;

phpadmin允许空密码登录:
/usr/share/phpmyadmin/libraries/config.default.php
找到2个nopassword的选项赋值true
$cfg['Servers'][$i]['nopassword'] = true;
$cfg['Servers'][$i]['AllowNoPassword'] = true;

系统软连接–快捷方式
ln -s sourcedir destdir

系统开机启动，查看用时
systemd-analyze
查看服务启动用时
systemd-analyze blame
禁止不用的服务开机启动
systemctl disable apt-daily.service

php 新增插件sudo apt-get install php7.1-sqlite

deepin 的自带终端可以通过deepin-terminal打开，配合一些其他命令可以当做快捷键使用，比如将deepin-terminal -e ssh root@xxx来快速打开自己的服务器

切换apache服务器的php版本，可以通过a2dismod php7.0 a2enmod php7.2实现
正式服务器可能由于配置了Deny all 所以不能用软连接的放项目

  • deepin系统升级15.7之后部分功能出现问题，首先在使用composer的时候提示缺失拓展文件，发现是自动升级了php到7.2，为了稳定回退7.1，方式是通过which php找出当前php命令快捷方式来源

  • 安装php7.1-curl提示安装libcurl3，安装后发现apache被自动删除了，需要重新安装

  • apache 发现莫名出现active(exited)，无法正常运行，在echo $PATH逐个找路径发现没有apache2，结论是apache服务在升级php插件的时候被删除，只能重装
    但是服务器配置仍然在，可能会出现DefaultRuntimeDir变量未定义的错误提示，可以运行source /etc/apache2/envvars来修复，然后运行apache2开启服务器，会提示缺少模块，用apt-cache search apache2 mod搜索逐个安装libapache2-mod-apreq2 libapache2-mod-perl2 libapache2-mod-php7.2

  • 近期在升级完系统之后发现php7.1-curl的安装需要libcurl3，但是切记安装了libcurl3之后会导致本身的libcurl4无效，同时会把apache2给删了；
    那删了之后自己再重新安装回apache2，但是重开机没多久就又出现php7.1-curl缺失的问题，应该是软件源上的版本不对应的缘故，只能等待修复，自己重新装包容易出现依赖问题

  • 系统升级之后显卡驱动没有设置好，会有明显的卡顿现象，需要自己通过深度显卡驱动重新配置一下!
  • 服务器配置vsftpd之后需要通过sftp://ip来进行连接

apache2 的泛解析配置

  • 参考

  • 启用vhosts_alias模块

  • 在配置站点文件的时候额外添加ServerAlias作为泛解析地址与VirtualDocumentRoot作为泛解析的站点目录
