symfony随记

2018-07-29

symfony4.1.1

  • composer create-project
  • 首次按照官方手册进行安装，无法通过installer安装(提示与4.x版本不兼容),可以换用composer安装
  • 发现命令php bin/console server:run（后面可以加自定义ip:port）无法运行，实际需要手动安装server组件composer require server --dev（参考）
  • 初始化的项目无法访问/config.php， 提示not found
  • /config.php访问问题得不到解决(中文文档的symfony版本为3),可以换用3.4.版本体验`composer create-project symfony/framework-standard-edition my_project_name “3.4.“`
  • 中文文档没有symfony4的入门文档，可以查看英文原版文档

第一个页面

  • 制定路由，除了在config/routes.yaml中定义外，还可以通过annotation 方式实现，需要安装插件composer require annotations,注意引用的时候第一行要有两个*才能正常访问,同时记得use Symfony\Component\Routing\Annotation\Route;( Annotations are the recommended way to configure routes.)(*注意的是，两种方法不能同时使用，否则annotation会失效)
  • web 端的toolbar composer require --dev symfony/profiler-pack
  • symfony/twig-bundle 有关模板渲染，xxx.html.twig模板文件放在templates下，命名可以参照controller/function
  • 渲染return $this->render('lucky/number.html.twig', [''=> $]) 需要controller extends AbstractController (FrameworkBundle)
  • 在模板文件中如果没有</body>则toolbar不会出现
  • 可以通过console自动生成controller,注意安装composer require symfony/maker-bundle --dev，之后bin/console make:controller
    ##Routing
  • symfony路由实现方式有多种annotations yaml xml php参考
  • locale 区域化的路由设置,注意不同区域缩写之间用,隔开
  • 会发现首页的访问会有404， 这是因为在symfony4中的路由没有配置首页GET \,可以通过bin/console debug:router查看，不必在意(我自己倒纠结了一整个上午)
  • 正则匹配 @Route("/blog/{page}", name="blog_list", requirements={"page"="\d+"})
    也可以@Route("/blog/{page<\d+>}", name="blog_list")，但是会降低可读性

    关于service的更多参考

  • bin/console debug:autowiring查看自动加载的服务
  • 如果按照文档$logger: '@monolog.logger.doctrine'会提示服务不存在，我们注释掉该行，debug:autowiring查看服务的名字，发现对于LoggerInterface的服务名为logger，修改为$logger: '@logger'后在控制器的函数上可以直接传递参数$logger而不必再申明LoggerInterface类型
  • entity的创建参考
    • 修改根目录下的.env，修改数据库信息DATABASE_URL
    • php bin/console doctrine:database:create 按照配置信息创建数据库
    • php bin/console make:entity
    • php bin/console make:migration 创建Migrations下的文件
    • php bin/console doctrine:migrations:migrate 执行Migrations下的未执行文件中的SQL语句
      如果需要新增字段
    • php bin/console make:entity
    • php bin/console make:migration
    • php bin/console doctrine:migrations:migrate
  • entity 保存之后可以通过命令php bin/console doctrine:query:sql 'SELECT * FROM product'查看结果
